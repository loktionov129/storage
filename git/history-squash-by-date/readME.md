# history-squash-by-date
## Скрипт `squashHistory.js` — объединение Git‑коммитов по периодам

Автоматизирует сквош (объединение) Git‑коммитов по периодам (год/месяц/день) с анализом тематики на основе Conventional Commits. Упрощает историю репозитория, сохраняя при этом осмысленные сообщения.


## Установка и требования


**Требования:**
* Node.js (версия 12+);
* Git (версия 2.0+).

**Использование:**
1. Сохраните скрипт как `squashHistory.js` в корне вашего Git‑репозитория.
2. Убедитесь, что репозиторий чист (нет незакоммиченных изменений).
3. Запустите скрипт с нужными параметрами.

## Параметры командной строки

| Параметр | Описание | Обязательный |
|---------|---------|-------------|
| `--strategy` | Стратегия группировки: `year`, `month` или `day` | Да |
| `--verbose` | Подробный вывод всех операций Git | Нет |

## Пример использования

### Исходное состояние репозитория

Предположим, у нас есть репозиторий с историей коммитов за 2023 год:

```bash
$ git log --oneline
a1b2c3d (HEAD -> main) fix: correct typo in README
9f8e7d6 feat: add user profile page
8e7d6c5 docs: update installation guide
7d6c5b4 feat: implement search functionality
6c5b4a3 fix: handle null values in API response
5b4a392 style: format code according to PEP8
...
(всего 45 коммитов за 2023 год)
```

### Выполнение команды

Запускаем скрипт для сквоша по месяцам:

```bash
node squashHistory.js --strategy month
```

### Процесс выполнения

1. Скрипт проверяет, что вы находитесь в Git‑репозитории и он чист.
2. Получает историю коммитов через `git log`.
3. Группирует коммиты по месяцам (например, все коммиты за октябрь 2023 года).
4. Анализирует тематику коммитов (находит наиболее частые ключевые слова типа `feat`, `fix` и т. д.).
5. Выводит сводку и запрашивает подтверждение:
   ```
   Found 45 commits to process
   Grouped into 12 periods
   Periods to process:
   - 2023-01
   - 2023-02
   ...
   - 2023-12

   Proceed with squash? (y/n):
   ```
6. Пользователь вводит `y`.
7. Для каждого месяца скрипт:
    * определяет доминирующую тему (например, «new features» для `feat`);
    * создаёт сообщение сквоша;
    * выполняет сквош через интерактивный ребаз;
    * логирует прогресс.

### Результат выполнения

**Состояние репозитория после выполнения:**


```bash
$ git log --oneline
c3d4e5f (HEAD -> main) squash: 4 commits for 2023-12 (bug fixes)
b2c3d4e squash: 5 commits for 2023-11 (new features)
a1b2c3d squash: 3 commits for 2023-10 (documentation updates)
...
(всего 12 коммитов — по одному на каждый месяц 2023 года)
```

**Финальный отчёт скрипта:**

```
==================================================
Commit merging completed.
Before: 45 commits
After: 12 commits
Reduced by: 33 commits
Processed periods:
- 2023-01
- 2023-02
...
- 2023-12
==================================================
```

## Примеры команд

**Сквош по годам:**
```bash
node squashHistory.js --strategy year
```

**Сквош по месяцам с подробным выводом:**
```bash
node squashHistory.js --strategy month --verbose
```

**Сквош по дням:**
```bash
node squashHistory.js --strategy day
```

## Обработка ошибок

Скрипт корректно обрабатывает следующие сценарии:
* **Запуск вне Git‑репозитория:** выводит ошибку «Error: Not in a Git repository» и завершает работу.
* **Незакоммиченные изменения:** выводит ошибку «Error: The repository has uncommitted changes» и завершает работу.
* **Неверная стратегия:** выводит ошибку с указанием допустимых значений (`year`, `month`, `day`).
* **Ошибки во время ребаза:** отменяет операцию, очищает временные файлы и выводит информативное сообщение.

## Особенности безопасности

* Перед началом сквоша запрашивается подтверждение у пользователя.
* Сообщения коммитов экранируются перед использованием в командах Git.
* Временные файлы очищаются после завершения или ошибки.
* Используется `GIT_SEQUENCE_EDITOR="true"` для кросс‑платформенности.

## Ограничения
* Работает только в локальных репозиториях.
* Требует чистого состояния репозитория (без незакоммиченных изменений).
* Не поддерживает сквош коммитов слияния (`merge commits`).
* Не обрабатывает коммиты без даты.
